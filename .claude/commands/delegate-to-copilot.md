# Delegate to Copilot

Generate a `copilot-plan.md` for Copilot CLI to execute. Supports both **implementation** and **review** tasks.

## ABSOLUTE RULES

1. **Output is ALWAYS `copilot-plan.md`** — NEVER create GitHub issues, NEVER use MCP tools, NEVER call GitHub API
2. **NEVER fix issues yourself** — only generate the plan file
3. **The plan must be self-contained** — Copilot has 160K context per agent, no access to this conversation

## Step 0: Check for Existing Plans

Before generating a new plan:
```bash
ls copilot-plan.md copilot-results.md review-results.md 2>/dev/null
```
If any of these files exist, warn the user: "Previous delegation artifacts found. These may be from an incomplete Copilot run." Ask whether to archive (rename with timestamp) or delete before proceeding.

## Determine Task Type

From $ARGUMENTS, determine whether this is:
- **Implementation**: "fix tests", "add feature X", "refactor Y" → generate implementation tasks
- **Review**: "review PR #N", "audit code", "review changes" → generate review tasks
- **Both**: "review and fix" → generate review tasks that include fix instructions

## Step 1: Capture Current State

```bash
git status
git log --oneline -10
git branch --show-current
```

For implementation tasks, also run:
```bash
# Project commands below
pnpm test 2>&1 | tail -50       
pnpm lint 2>&1 | tail -20   
pnpm typecheck 2>&1 | tail -20  
```

For review tasks, identify the files to review:
```bash
# For specific PRs:
gh pr view <NUMBER> --json files -q '.files[].path'
gh pr diff <NUMBER>
# For branch changes:
git diff --name-only $(git merge-base HEAD main)..HEAD
```

## Decision Gate

If all verification commands pass AND there is no actionable work based on $ARGUMENTS:
- Report: "Starting state is already clean — nothing to delegate"
- Do NOT create copilot-plan.md
- STOP

## Step 2: Generate copilot-plan.md

### For Implementation Tasks:

```markdown
# Copilot Execution Plan
Generated by Claude Code on [date]
Branch: [branch]
Type: IMPLEMENTATION

## Starting State
[test/lint/typecheck results]

## Objective
[what needs to be done]

## Acceptance Criteria
- [ ] All tests pass
- [ ] Lint clean
- [ ] Typecheck clean
- [ ] All 25 reviewers grade >= 95/A+

## Execution Instructions
1. Read this entire plan
2. Execute tasks by phase (sequential phases, parallel within)
3. Run verification gate after all tasks
4. Invoke @review-pipeline
5. Create PR with grades table
6. Output results to copilot-results.md

## Phase 1: [Name]
### Task 1.1: [Title]
**Objective:** [what and why]
**Files to modify:** [exact paths]
**Files to read (context):** [reference files]
**Implementation:** [step-by-step with code snippets from codebase]
**Acceptance:** [specific conditions]
```

### For Review Tasks:

```markdown
# Copilot Execution Plan
Generated by Claude Code on [date]
Branch: main
Type: REVIEW + FIX

## Objective
Review the following code and fix all findings until all 25 reviewers grade >= 95/A+.

## What to Review
[List of PRs, files, or changes to audit]

### PR #[N]: [title]
**Files:**
- `path/to/file1.ts`
- `path/to/file2.ts`
**PR description:** [summary of what the PR did]

## Execution Instructions
1. Create a new branch: `fix/copilot-audit-[date]`
2. Read ALL files listed above
3. Invoke @review-pipeline on these files
4. Fix ALL findings (BLOCK + WARN + INFO) from all 25 reviewers
5. Re-run @review-pipeline until all grade >= 95/A+
6. Run verification gate (tests + lint + typecheck)
7. Create PR with grades table
8. Output results to copilot-results.md

## Review Focus
[Specific areas to focus on, based on what the PRs did]

## Acceptance Criteria
- [ ] All 25 reviewers grade >= 95/A+
- [ ] All tests pass
- [ ] Lint clean
- [ ] Typecheck clean
- [ ] Fix PR created and ready for merge
```

## Step 3: Validate

Before saving, verify:
1. **Self-contained**: No references to "this conversation" or context Copilot won't have
2. **Token budget**: No task block exceeds ~100K tokens
3. **File lists complete**: Every file that needs attention is listed with exact paths
4. **Patterns included**: Code snippets from codebase, not abstract descriptions

## Step 4: Save and Report

```bash
# Write copilot-plan.md to project root
git add copilot-plan.md
git commit -m "chore: copilot execution plan"
git push
```

Tell the user:
- Plan saved to `copilot-plan.md`
- Number of tasks/files
- "Run `@plan-executor execute copilot-plan.md` in Copilot CLI"
